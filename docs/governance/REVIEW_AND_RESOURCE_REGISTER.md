# 海豚社区App审核与资源登记

**作者：Manus AI**  
**用途：** 这份文档用于记录后续开发中的设计审核、功能审核、发布审核，以及未来可能需要用户提供的秘钥、账号、权限与三方服务接入信息。

## 一、审核机制

后续开发不会采用“做完再看”的粗放方式，而是以固定审核节点推进。也就是说，每一个关键里程碑都必须经过**设计审核、功能审核、链路审核与交付审核**四类检查，其中任何一类不过关，都要回流成返修任务，而不是直接推进到下一个阶段。

| 审核类型 | 审核目标 | 审核方式 | 通过标准 |
|---|---|---|---|
| 设计审核 | 确认页面是否符合极简苹果感与高端金融质感 | 截图走查、组件对照、排版复核 | 风格统一、层级清晰、卡片可信、无视觉噪音 |
| 功能审核 | 确认功能是否真实可用或有可靠 mock 兜底 | 手动走链路、自测、接口检查 | 不出现断链、无反馈、误导用户的行为 |
| 链路审核 | 确认跨页面与对话上下文是否连续 | 从入口到执行回执全链路验证 | 用户不迷路，关键动作有回执 |
| 交付审核 | 确认是否达到可预览、可演示、可构建状态 | 测试、截图、构建检查 | 结果可复现，可交付给你验收 |

## 三、资源登记规则

我只会在**确实无法继续独立推进**时，才向你索取资源。所有需求都要写清楚用途、风险边界与最小必要范围，不会无条件向你要一堆秘钥。

| 资源类别 | 当前是否需要 | 未来触发条件 | 典型用途 |
|---|---|---|---|
| LLM 推理秘钥 | 否 | 接入真实生产级对话与 Agent 能力时 | 自然语言理解、结构化回复、Agent 编排 |
| 邮件验证码服务 | 否 | 真实注册/登录验证码需要对外发送时 | 验证码发送与登录链路 |
| OKX / 钱包 / 交易秘钥 | 否 | 真正执行转账、Swap、赚币等交易动作时 | 真实链路执行与账户能力 |
| 推送通知服务 | 否 | 需要任务提醒、执行回执通知时 | 消息提醒、后台任务通知 |
| EAS / 发布权限 | 否 | 准备构建正式安装包或发布测试包时 | APK / AAB 构建与分发 |
