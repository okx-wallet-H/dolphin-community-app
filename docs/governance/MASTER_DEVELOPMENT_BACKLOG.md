# 海豚社区App主开发清单

**作者：Manus AI**  
**仓库：** `okx-wallet-H/dolphin-community-app`  
**本清单定位：** 这是后续开发的**唯一主清单**。我后续所有开发、验证、记录、返修、交付与审核，都以这份文档为总表，不再沿用原仓库历史遗留的零散任务项。

## 一、产品核心与开发原则

海豚社区App后续的产品核心已经明确，不再是“做一个普通钱包界面”，而是做一个**以对话式交互为主入口的 AI Agent 钱包产品**。用户通过自然语言表达意图，系统负责理解、澄清、展示结构化卡片，并在对话上下文中完成资产查询、策略推荐、交易执行、赚币配置、社区发现与任务自动化。界面层的目标不是堆砌功能模块，而是让用户在一条连续的对话链中完成整个操作过程。

因此，后续开发必须同时满足三条标准。第一，**交互必须对话优先**，所有关键能力都要能回到聊天主线程里完成。第二，**卡片必须高级且可信**，所有 AI 返回的结构化结果都要具备金融产品应有的层级、数字排版、风险提示与动作反馈。第三，**工程必须持续可维护**，每一项功能都必须在任务清单、执行台账和验收记录中留下痕迹，避免后续失忆、重复开发或回归失控。

## 二、主开发工作流

后续开发按照“定义、实现、验证、审核、记录、交付”六步闭环推进。每一个任务在真正编码前，都必须先进入主清单与执行台账；每完成一个关键节点，都必须补上自测结果、截图或测试结论；遇到需要你提供外部资源时，我再单独向你发起请求，其余部分我会持续自主推进。

| 工作流阶段 | 我负责的动作 | 输出物 | 是否需要你参与 |
|---|---|---|---|
| 定义 | 拆解需求、归类优先级、明确验收口径 | 主开发清单、任务台账 | 一般不需要 |
| 实现 | 编码、重构、补组件、接接口、联调 | 代码提交、组件、页面、服务 | 一般不需要 |
| 验证 | 自测、类型检查、测试、截图、走查 | 测试记录、验证记录、截图 | 一般不需要 |
| 审核 | 设计复核、交互复核、链路复核 | 审核结论、返修任务 | 一般不需要 |
| 资源请求 | 仅在缺少秘钥、账号、白名单、支付权限时发起 | 资源请求登记 | 需要时才需要 |
| 交付 | 阶段汇报、可预览版本、安装包、文档 | 结果说明、附件、发布产物 | 需要你确认方向 |
