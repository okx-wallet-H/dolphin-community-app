# H Wallet 行情与交易阻塞根因说明

## 结论

你现在看到的两个问题，**不是同一个根因**。

第一，**多数行情查不到**，核心原因是当前实现仍是**白名单式支持**，并没有把“任意币种查询”真正做通。第二，**交易无法真正执行**，核心原因是当前会话里**没有恢复出真实 Agent Wallet 地址与登录态**，所以交易链路会在真正提交前被前端直接拦截。

## 一、为什么多数行情都查不到

当前代码里，行情查询不是“输入任意 symbol 就去全市场查”，而是写死了有限映射。

| 位置 | 当前支持范围 | 影响 |
|---|---|---|
| `lib/_core/api.ts` 中 `MARKET_PRICE_TARGET_MAP` | `BTC`、`ETH`、`SOL`、`USDT`、`OKB` | 不在映射里的币，直接返回“不支持查询该行情” |
| `lib/_core/api.ts` 中 `getMarketSnapshotByMcp` 的 `symbolToAddress` | `BTC`、`ETH`、`SOL`、`OKB`、`BNB`、`SUI` | 行情页本身也只覆盖少数主流币 |
| `app/(tabs)/market.tsx` 中 `MARKET_BASE` | `BTC`、`ETH`、`SOL`、`BNB`、`SUI` | 页面只渲染固定几张主流币卡片，不是全市场搜索 |
| `server/_core/chat-ai.ts` 中价格兜底映射 | `BTC`、`ETH`、`SOL`、`OKB` | 聊天问价也仍偏白名单逻辑 |

这意味着当前产品实际上只做通了“**少数主流币固定查询**”，并没有做通“**其他币都能查**”。所以你问很多别的币，前端并不是查询失败后回退，而是**从设计上就没有完整支持**。

## 二、为什么交易无法交易

本轮运行时实测里，一句话交易“帮我购买3U的BNB”已经能被识别成交易意图，但会被下面这个前置条件挡住：

| 检查点 | 当前实现 | 结果 |
|---|---|---|
| 聊天页本地快路径解析 | 已生效 | 能识别 `3 USDT -> BNB` |
| 钱包会话恢复 | 依赖 `AsyncStorage` 或 `getMe()` 恢复 `evmAddress / solanaAddress` | 当前 Web 预览没有恢复出来 |
| 真实交易执行前置校验 | 若没有真实钱包地址，直接中断 | 不会继续到确认、广播、回执 |
| 服务端执行接口 | `/api/onchain/execute` 需要登录态 | 未恢复会话时即使前端继续发，也会被鉴权挡住 |

聊天页里已经写了明确拦截逻辑：只要当前 `wallet` 不存在、`mockMode` 为真，或者 `evmAddress / solanaAddress` 都为空，就直接提示“当前会话还没恢复出真实 Agent Wallet 地址，请重新登录一次”。

所以现在的交易问题，本质上不是“识别不了交易句子”，而是：

> **句子能识别，但没有真实 Agent Wallet 会话，执行链路就无法继续。**

## 三、再往下看，还有一个结构性限制

即使登录态恢复了，当前交易快路径本身也仍然是**有限币种白名单**，不是全币种通用。

| 位置 | 当前支持 |
|---|---|
| EVM 交易候选 | `USDT`、`USDC`、`ETH`、`BTC`、`BNB` |
| Solana 交易候选 | `USDT`、`USDC`、`SOL` |

因此，哪怕登录态恢复后，很多长尾币或未配置地址的币，仍然不会真正走到可执行交易。

## 四、用一句话总结

| 问题 | 直接原因 |
|---|---|
| 其他行情查不到 | 当前行情查询仍是固定币种映射，不是全市场查询 |
| 交易无法执行 | 当前会话没有恢复出真实 Agent Wallet 地址，前端在执行前就拦截了 |
| 某些币即使恢复登录也未必能交易 | 交易快路径本身也只有少数币种白名单 |

## 五、下一步应怎么修

下一步应该分两层修，而不是只改页面文案。

第一层，要把**登录态与钱包恢复链路**修稳，确保 `getMe()`、会话 cookie / session token、`AsyncStorage` 钱包快照能在 Web 和真机上都恢复出真实 `evmAddress` 与 `solanaAddress`。这一步不通，所有真实交易都会停在预执行前。

第二层，要把**行情和交易从白名单逻辑升级为可检索逻辑**。也就是不要再只靠手写 `BTC/ETH/SOL/...` 映射，而是改成“symbol 搜索 -> 获取合约地址 / instId -> 查询价格 / 生成交易参数”的动态链路。否则现在只是假装支持自然语言，实际上仍然只能处理少数预埋币种。
