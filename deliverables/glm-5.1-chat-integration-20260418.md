# 海豚社区 App 接入 GLM 5.1 结论与当前落地情况

**结论先说**：GLM 5.1 可以直接接入海豚社区现有聊天主链，而且适合作为当前的**自然语言主交互模型**。原因在于智谱官方文档已经明确给出了 **OpenAI 风格的 Chat Completions 接口**、`Bearer` 鉴权方式、标准 `messages` 请求体、结构化 JSON 返回能力，以及 `Function Call` / `MCP` / 流式输出等能力，这与海豚社区现在“模型负责理解用户意图，OKX Onchain OS 负责执行钱包、交易、行情、赚币四大技能”的架构是匹配的。[1]

从官方文档可直接提取出海豚社区当前最关键的接入契约：**请求地址**为 `https://open.bigmodel.cn/api/paas/v4/chat/completions`，**请求头**使用 `Authorization: Bearer <API Key>`，**模型名**为 `glm-5.1`，并且示例里已经展示了 `response_format`、`thinking`、`stream` 等控制项的使用方式。[1] 这意味着海豚社区没有必要额外重做一套模型协议，只需要把现有聊天入口从“固定 OpenAI 专用写法”收敛成“可配置的 OpenAI 兼容接口”，即可把 GLM 5.1 接进来。

| 项目 | 官方结论 | 对海豚社区的意义 |
|---|---|---|
| 模型名 | `glm-5.1` [1] | 可直接作为聊天主模型默认值 |
| 接口地址 | `https://open.bigmodel.cn/api/paas/v4/chat/completions` [1] | 现有聊天主链可直接走兼容式 POST 调用 |
| 鉴权方式 | `Authorization: Bearer your-api-key` [1] | 与现有服务端密钥注入方式兼容 |
| 请求结构 | `messages` 数组 [1] | 可直接复用现有意图识别 prompt 结构 |
| 结构化输出 | 支持 JSON 等结构化输出 [1] | 适合返回 `action/confidence/reply` 等结构化意图 |
| 工具能力 | 支持 Function Call、MCP [1] | 后续可继续向 OKX 四大技能编排升级 |
| 上下文能力 | 200K 上下文窗口 [1] | 更适合承接海豚社区长对话与多轮意图补参 |

本轮我已经按这个契约把代码层做了第一轮正式收敛。核心改造不是“在项目里硬塞一个 GLM 特判”，而是把原本写死的模型层抽象成**通用主模型配置入口**，让海豚社区后端能够通过环境变量切换到 GLM 5.1，同时保持未来继续兼容其他 OpenAI 风格模型的能力。这样做的好处是：海豚社区以后若要更换模型，只改配置，不必再次改聊天主链。

| 已完成改造 | 说明 | 文件 |
|---|---|---|
| 增加统一 LLM 配置 | 新增 `LLM_API_URL`、`LLM_API_KEY`、`LLM_MODEL`，并兼容 `GLM_*` 命名 | `server/_core/env.ts` |
| 改造通用模型调用层 | 统一支持 OpenAI 兼容接口，默认可直连 GLM 5.1 | `server/_core/llm.ts` |
| 替换聊天主链模型调用 | 将原有 OpenAI 专用聊天意图调用切换为复用统一 LLM 层 | `server/_core/chat-ai.ts` |
| 修正前端类型契约 | 把聊天意图来源标识从 `openai` 改为通用 `llm` | `lib/_core/api.ts` |
| 打通 serverless 聊天入口 | 将 `/api/chat` 与 `/api/chat-ai/intent` 改为真实调用聊天意图主链，而非不可用占位 | `api/chat/index.ts` |
| 更新环境示例 | 明确 GLM 5.1 的配置写法和默认地址 | `.env.example` |

从海豚社区业务架构角度看，本轮接入后的分层已经比较清晰。**GLM 5.1 只负责理解用户自然语言、识别意图、返回结构化槽位**；而真正的钱包、交易、行情、赚币执行依旧走已经接好的 OKX Agent Wallet / Onchain OS API。这种拆法能保证模型只做“理解与编排”，不会侵入 OKX 的执行层，也完全符合你之前反复强调的原则：

> **OKX 已经把底层能力开发好了，海豚社区只做对接、编排和产品封装。**

| 分层 | 现在的职责 |
|---|---|
| **GLM 5.1 模型层** | 识别 `market / asset / swap / earn / profit / deposit / general` 意图，输出结构化 JSON |
| **海豚社区聊天主链** | 把用户自然语言与钱包上下文传给模型，并把结构化结果路由到四大技能 |
| **OKX 四大技能执行层** | 真正完成钱包、交易、行情、赚币能力调用 |
