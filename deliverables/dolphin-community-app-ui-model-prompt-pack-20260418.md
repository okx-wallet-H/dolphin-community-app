# 海豚社区 App：UI 设计师模型提示任务包

**作者：Manus AI**  
**日期：2026-04-18**  
**冻结代码基线：`c3ef22b`**

## 一、使用方式

这份文档不是开放式创意简报，而是给 UI 设计师模型直接使用的 **受约束任务包**。使用时必须先阅读“全局边界”和“统一视觉要求”，然后严格按页面顺序执行。模型不得自行改写产品结构，也不得用营销页思路覆盖当前可测试 MVP 的页面职责。

模型输出应优先服务于前端可落地性。也就是说，页面方案必须能够回到现有 React Native 页面结构中实现，不能依赖全新导航、额外功能模块或未定义的复杂动效。

## 二、全局边界

| 维度 | 固定要求 |
|---|---|
| 产品角色 | 当前是可测试 MVP，不是开放式品牌改版 |
| 页面职责 | 钱包页负责资产总览，币种明细页负责账号明细，聊天页负责意图承接和执行入口，赚币页只展示真实产品 |
| 结果归属 | 交易结果沉淀到账号明细，不以聊天页作为主追单中心 |
| 数据真实性 | 不允许出现 demo 卡片、假数据、演示型占位模块 |
| 实现方式 | 方案必须可映射到现有前端代码结构，不允许要求整页推倒重写 |
| 设计语言 | 统一沿用当前产品已使用的淡紫色金融科技风格，强调克制、秩序、高级感 |

## 三、统一视觉要求

本次设计语言应强调 **高端金融产品质感**，但不能走过度炫技路线。页面应该给人“可信、成熟、轻盈、克制”的感觉，而不是“概念稿、营销页、AI 炫技图”。

| 项目 | 方向要求 |
|---|---|
| 视觉气质 | 高级、克制、可信、柔和、有秩序 |
| 信息层级 | 主路径最强，次要模块明显降噪 |
| 材质感 | 可以有轻玻璃感、浅渐变、柔和阴影，但不能浮夸 |
| 色彩 | 以白色、淡紫、浅灰为主，强调金融科技感 |
| 留白 | 强调呼吸感与阅读节奏，避免信息块拥挤 |
| 控件 | 按钮、标签、资产行、状态卡要精致但不过度复杂 |

## 四、明确禁止项

| 禁止项 | 说明 |
|---|---|
| 改主路径 | 不允许改动页面间职责和入口流向 |
| 新增业务模块 | 不允许新增未定义的浮层、Tab、结果面板、统计中台 |
| 加演示数据 | 不允许加入任何 mock、demo、虚构收益或假记录 |
| 变成营销风 | 不允许做成电商页、海报页、Web3 炫酷概念稿 |
| 过度装饰 | 不允许大面积霓虹、复杂 3D、重颗粒特效 |

## 五、统一输出要求

模型为每一页输出方案时，必须同时给出三个层面的内容：第一，页面要解决的核心问题；第二，建议的视觉收口方向；第三，哪些点必须保留不动，以便前端能快速落地。输出风格必须专业，不能只给一句泛泛的设计描述。

## 六、页面任务包

### 1. 钱包页

钱包页是第一优先级。目标不是更花，而是更像正式金融产品首页。页面应突出“总资产估值”和“链上资产列表”，弱化中段辅助模块的抢眼程度，让用户一眼知道主任务是看资产和点进币种详情。

| 项目 | 任务要求 |
|---|---|
| 主目标 | 让钱包页更像资产中台首页 |
| 重点优化 | Hero 质感、地址胶囊、资产列表分组、资产行点击态、分区节奏 |
| 次要模块 | 钱包安全、自动任务需要降噪，不能与主资产区争抢焦点 |
| 不可改动 | 资产列表点击进入币种明细；页面仍保留现有主要模块结构 |

**钱包页模型提示词：**

> 请基于一个移动端金融科技 App 的钱包页，生成高端、克制、可信的 UI 收口方案。页面主任务是展示总资产估值与链上资产列表，用户点击资产项会进入该币种的账号明细页。请强化 Hero 区的金融产品质感，优化地址胶囊、资产分组标题、资产行间距和点击态反馈；同时弱化“钱包安全”和“自动任务”模块的视觉噪音，使其保持辅助角色。整体风格请使用白色、淡紫、浅灰体系，强调高级感与秩序，不要增加新的业务入口，不要改动页面主路径，不要加入演示数据，不要做成营销概念稿。

### 2. 币种明细页

币种明细页是第二优先级。这里不是一般详情页，而是账号明细中心。设计重点是提升账本感和秩序感，让用户自然理解“余额—估值—记录状态—资产流向”的阅读顺序。

| 项目 | 任务要求 |
|---|---|
| 主目标 | 做成正式账本页，而不是临时详情页 |
| 重点优化 | Hero 的币种身份感、数值排版、记录卡状态标签、流向区、摘要区 |
| 状态场景 | 正常记录态、空状态、加载态、失败态都要统一且可信 |
| 不可改动 | 结果归属仍在账号明细，不回到聊天页追单 |

**币种明细页模型提示词：**

> 请基于一个移动端金融科技 App 的币种账号明细页，生成高端、稳重、有秩序的 UI 收口方案。页面主任务是展示某个币种的余额、估值和真实交易记录，交易记录可能包含成功、失败和处理中状态。请重点优化头部 Hero、状态标签、资产流向卡片、摘要与详情信息的阅读顺序，让页面更像正式账本而不是普通详情页。整体风格应延续白色、淡紫、浅灰体系，强调可信、沉淀、金融专业感。不要把结果入口改回聊天页，不要新增新的业务功能，不要加入演示数据。

### 3. 聊天页交易卡片

聊天页第三优先级。交易卡片的职责是承接意图、提示当前状态、提供执行动作，不是完整交易控制台。因此设计应减少控制台感、减少信息过载，做出“服务单式”的清晰感。

| 项目 | 任务要求 |
|---|---|
| 主目标 | 降低工具台感，增强服务感 |
| 重点优化 | 卡片头部层级、状态区轻量化、按钮主次、说明文案轻重 |
| 信息原则 | 保留必要信息，但不堆砌技术状态和过多解释 |
| 不可改动 | 不新增复杂流程步骤，不把聊天页变成主追单中心 |

**聊天页模型提示词：**

> 请基于一个移动端 AI 助手聊天页中的交易卡片，生成更克制、更成熟的 UI 收口方案。卡片用于承接兑换和转账意图，只负责简洁展示关键信息、当前状态和执行入口，不是完整交易控制台。请优化卡片头部、状态区、按钮层级和留白，使其更像服务单而不是操作台。整体风格保持白色、淡紫、浅灰体系，强调服务感、可信感和轻量化。不要增加新按钮分叉，不要加入复杂步骤条，不要把聊天页做成主结果中心。

### 4. 赚币页

赚币页第四优先级。当前页面已经去掉 demo fallback，下一步重点是让“真实投资产品列表”成为真正主角。辅助信息可以存在，但必须退居第二层。

| 项目 | 任务要求 |
|---|---|
| 主目标 | 让真实产品卡片成为页面主角 |
| 重点优化 | Hero 收敛、辅助洞察降噪、产品卡一致性、空状态可信度 |
| 风格方向 | 更像真实产品选择页，而不是策略营销页 |
| 不可改动 | 不允许恢复 demo 卡片、假收益、占位策略 |

**赚币页模型提示词：**

> 请基于一个移动端金融科技 App 的赚币页，生成克制、可信、非演示化的 UI 收口方案。页面只展示真实投资产品，无产品时直接显示空状态。请优化页面头部、Hero、辅助信息区和产品卡片，使真实产品列表成为视觉主角，同时保留金融产品页应有的稳定感和高级感。整体风格保持白色、淡紫、浅灰体系，不要夸张收益表达，不要引入 demo 卡片、假策略或营销化元素。

## 七、设计产出如何交给前端

设计师模型给出方案后，不能直接视为最终稿，而是要先经过我这一层复核。我会先过滤掉任何改变主路径、增加无效功能、脱离现有代码结构的部分，再把真正可落地的样式和结构建议翻译成前端任务。

| 步骤 | 责任方 | 产出 |
|---|---|---|
| 1 | UI 设计师模型 | 单页收口方案 |
| 2 | Manus AI | 方向复核、去噪、前端翻译 |
| 3 | 前端实现 | 基于现有代码逐页落地 |
| 4 | Manus AI | 再次复核并决定是否进入下一页 |

## 八、执行顺序

本轮执行顺序已经冻结，不允许跳页。必须先做钱包页，再做币种明细页，再做聊天页交易卡片，最后才是赚币页。

> 先钱包页，后币种明细页；先单页通过，再进入下一页。这是为了确保设计和前端不是并行失控，而是在我主导下形成闭环协作。
