# 海豚社区 App：UI 公共文件索引

**作者：Manus AI**  
**日期：2026-04-18**

## 一、说明

本索引用于统一管理当前海豚社区 App 的 UI 协作材料，便于设计师、前端与项目成员在同一套文件基础上推进工作。所有文件均围绕当前冻结的 MVP 页面边界、逐页审看机制与设计落地约束展开，不允许脱离这些文件单独发散。

## 二、文件清单

| 文件 | 用途 | 适用对象 |
|---|---|---|
| `dolphin-community-app-ui-review-brief-20260418.md` | 冻结产品边界与 UI 不可改动项 | 设计师、前端、产品 |
| `dolphin-community-app-ui-self-review-round2-20260418.md` | 我主导完成的第二轮页面自审结论 | 设计师、前端 |
| `dolphin-community-app-ui-designer-kickoff-20260418.md` | 正式开工说明与逐页任务拆解 | 设计师 |
| `dolphin-community-app-ui-review-workflow-20260418.md` | 逐页评审机制与交付节奏 | 设计师、前端、项目负责人 |
| `dolphin-community-app-ui-model-prompt-pack-20260418.md` | 可直接投喂 UI 设计师模型的提示任务包 | AI 设计协作、我本人 |

## 三、使用顺序

后续协作必须遵循统一顺序。任何设计方案都要先读边界说明，再看我的自审结论，然后按开工说明与评审机制逐页推进。如果采用 UI 设计师模型，则必须使用提示任务包，并且模型产出仍需先经过我复核，才能翻译为前端实现任务。

| 顺序 | 文件 | 作用 |
|---|---|---|
| 1 | Review Brief | 锁定边界，防止做偏 |
| 2 | Self Review Round 2 | 了解当前问题与已收口项 |
| 3 | Designer Kickoff | 按页开工 |
| 4 | Review Workflow | 按流程提交与审核 |
| 5 | Model Prompt Pack | 用于 AI 设计师模型协作 |

## 四、结论

这套文件就是当前项目 UI 协作的公共基线。后续无论是人工设计师还是 UI 设计师模型，都必须基于这套文件工作，由我统一审核收口。
