# 海豚社区 App：UI 设计师开工说明

**作者：Manus AI**  
**日期：2026-04-18**  
**当前代码基线：`f873ecb`**

## 一、开工前提

当前海豚社区 App 已完成一轮 MVP 功能收口与一轮由我主导的 UI 自审。接下来允许 UI 设计师进入工作，但角色定位必须非常明确：这不是重新定义产品，而是基于现有代码和既定链路做 **高端质感优化、视觉层级整理、逐页审看修正**。

也就是说，UI 设计师的任务是把现有页面做得更成熟、更统一、更像正式产品，而不是发散重构。产品职责、页面入口、执行主路径已经冻结，不能再由设计端重新解释。

## 二、已冻结的产品边界

| 模块 | 已冻结结论 | 严禁改动项 |
|---|---|---|
| 钱包页 | 资产总览是主入口，点击资产项进入币种明细页 | 不允许改成弹层，不允许取消资产点击入口 |
| 币种明细页 | 成功与失败记录都沉淀到账号明细 | 不允许把主结果入口改回聊天页 |
| 聊天页交易卡片 | 只承接意图、轻提示和执行动作 | 不允许做成复杂控制台或追单中心 |
| 赚币页 | 只展示真实投资产品，无结果时直接空状态 | 不允许恢复 demo 卡片、假数据或演示策略 |
| 设计系统 | 统一回到现有设计令牌体系 | 不允许另起一套配色、字号、圆角、阴影规则 |

## 三、设计师本轮工作目标

UI 设计师本轮要完成的不是整包重绘，而是严格按顺序做逐页收口。我要求的核心结果只有三个：第一，提升页面的 **高级感**；第二，强化跨页的 **统一性**；第三，在不改变功能职责的前提下改善 **信息阅读顺序**。

| 页面 | 本轮目标 | 审看关键词 |
|---|---|---|
| 钱包页 | 做成正式金融产品首页 | 克制、总览感、资产中心 |
| 币种明细页 | 做成正式账本页 | 秩序、可信、沉淀感 |
| 聊天页交易卡片 | 从工具台收成服务卡片 | 轻量、明确、非控制台 |
| 赚币页 | 让真实产品列表成为主角 | 可信、干净、非演示化 |

## 四、逐页任务拆解

### 任务 1：钱包页

钱包页是第一优先级。设计师要先审钱包页，不允许跳页。当前钱包页已经具备主链路，但还需要继续压缩次要模块的存在感，让用户视线优先落在资产总览和链上资产列表上。

| 项目 | 设计师要做 | 不允许做 |
|---|---|---|
| Hero 区 | 提升金融首页气质，优化留白、标题重心、地址胶囊的精致度 | 不允许新增功能入口 |
| 链上资产列表 | 优化资产行间距、点击态、箭头和分组节奏 | 不允许改掉“点击资产进明细”的主路径 |
| 钱包安全 / 自动任务 | 可以弱化视觉权重、降低噪音感 | 不允许提升为主视觉模块 |

### 任务 2：币种明细页

币种明细页是第二优先级。它是后续 APK 可测性的关键页面，所以不能只做“好看”，必须做成真正的账号明细承接页。

| 项目 | 设计师要做 | 不允许做 |
|---|---|---|
| Hero 卡 | 优化币种身份感、估值与余额的排版秩序 | 不允许加新的业务按钮 |
| 账本记录卡 | 优化状态标签、流向关系、摘要与详情顺序 | 不允许把结果链路迁回聊天页 |
| 空状态 / 加载态 | 提升可信度和一致性 | 不允许增加演示型示意内容 |

### 任务 3：聊天页交易卡片

聊天页第三优先级。目标是继续降低控制台感，强化“服务接待台”气质。交易卡片应该像一张清晰的服务单，而不是一块信息面板。

| 项目 | 设计师要做 | 不允许做 |
|---|---|---|
| 卡片头部 | 优化图标、标题、链路标签的层次 | 不允许改动卡片职责 |
| 状态区 | 可以更轻、更稳、更克制 | 不允许增加复杂流程步骤 |
| 按钮区 | 优化主次按钮层级和间距 | 不允许加入新的操作分叉 |

### 任务 4：赚币页

赚币页第四优先级。当前方向已经对了，但还需要进一步清理首屏噪音，让真实投资产品卡片成为真正主角。

| 项目 | 设计师要做 | 不允许做 |
|---|---|---|
| Hero 区 | 压缩说明型表达，强调真实结果与可信感 | 不允许夸大收益感或做营销页风格 |
| 辅助信息区 | 可以继续轻量化洞察卡和快捷入口 | 不允许新增假模块 |
| 产品卡片 | 提升一致性与精致度 | 不允许恢复 mock、demo 或占位卡 |

## 五、设计师工作顺序与提交流程

设计师必须按以下顺序推进，不能跨页并行发散。每完成一页，只能先交给我审看，得到我确认后才能进入下一页。

| 顺序 | 页面 | 交付物 | 审看人 |
|---|---|---|---|
| 1 | 钱包页 | 问题清单 + 修正建议稿 | 先由我审核 |
| 2 | 币种明细页 | 问题清单 + 修正建议稿 | 先由我审核 |
| 3 | 聊天页交易卡片 | 问题清单 + 修正建议稿 | 先由我审核 |
| 4 | 赚币页 | 问题清单 + 修正建议稿 | 先由我审核 |

## 六、给设计师的直接口径

> 当前仓库已经有可运行的 MVP 结构。你不需要重新定义产品，也不能自由增加模块。请严格围绕钱包页、币种明细页、聊天页交易卡片、赚币页四个页面，按既有边界做高端质感优化。每次只审一页，先指出问题，再给修正建议，必须先经我确认，才进入下一页。

## 七、我这边的评审规则

我会以以下标准审核 UI 设计师的产出，任何一条不满足都不会直接放行。

| 评审维度 | 通过标准 |
|---|---|
| 产品边界 | 不改变页面职责和主路径 |
| 设计统一性 | 必须回到现有令牌体系，不另起风格 |
| 高级感 | 通过留白、层级、材质感提升，而不是装饰堆叠 |
| 可落地性 | 必须能回到现有代码结构中实现 |
| 迭代纪律 | 必须逐页推进，不允许整包发散 |

## 八、开工后的下一步

设计师现在可以正式开工，但只允许从 **钱包页** 开始。我会继续担任总负责人，先接收其逐页反馈，再决定哪些建议进入代码实现，确保页面不会被“做漂亮但做偏”。
