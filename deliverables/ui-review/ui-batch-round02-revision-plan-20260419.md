# 海豚社区 App：四页界面整包 Round 2 回改计划

**作者：Manus AI**  
**日期：2026-04-19**  
**身份：UI 设计师回改执行计划**

本文件用于把总负责人在 **Round 1 整包统一审核** 中回写的意见，收敛成下一轮可直接执行的回改计划。Round 2 不重新发散方向，也不重做页面职责，而是在现有统一视觉方向上进行**系统性减法**与**语义纠偏**，然后再按仓库链路提交新一轮整包稿。[1]

## 一、Round 2 总目标

本轮四页回改的共同目标有三项。第一，继续沿用已经建立起来的白底、浅紫、浅灰、圆角卡片语言，不推翻现有方向。[1] 第二，全面清理正式页面中的备注式、解释式、交付式、安抚式文字，使页面真正符合“简洁、清爽、正式产品感”的标准。[1] 第三，优先修正币种明细页中会损害产品可信度的语义和字段错误，确保资产名称、金额、单位、状态、链信息自洽。[1] [3]

## 二、整包统一执行项

| 优先级 | 统一执行项 | 执行方式 |
|---|---|---|
| P0 | 删除备注式与解释式文字 | 删除来源说明、原则说明、交付式提示、重复解释句，正式页面只保留必要产品文案 [1] |
| P0 | 修正语义错误 | 币种名称、链名、余额、估值、状态、订单字段全部重新核对，禁止单位混写和资产错配 [1] [3] |
| P1 | 压缩次要模块 | 非主任务模块统一降权，不在首屏与主内容争主视线 [1] |
| P1 | 统一信息密度 | 减少“一句话重复说两遍”的情况，让四页更像正式产品而不是说明书 [1] [4] |
| P1 | 收口正式产品语言 | 所有标题、说明句、按钮旁辅助文案改为更短、更稳、更产品化的表达 [1] |

## 三、分页回改重点

| 页面 | 回改重点 | Round 2 目标 |
|---|---|---|
| 钱包页 | 压缩首屏下半区次要模块，弱化来源说明，减轻分组头部文字 | 让“总资产 + 链分组资产列表”成为绝对主角 [2] |
| 币种明细页 | 优先修正资产名称、链信息、金额与单位的语义错误，删除解释句 | 先恢复可信度，再做视觉收口 [3] |
| 聊天交易卡片页 | 删除重复语义，压缩系统播报式标题，减少按钮周边辅助文字 | 让卡片更轻、更像服务确认单 [4] |
| 赚币页 | 弱化顶部 Banner 与底部原则说明，压缩产品说明句，收紧标签与按钮权重 | 去掉展示稿痕迹，收成正式产品列表页 [5] |

## 四、执行顺序

虽然这次采用整包回改，但内部执行仍保持清晰顺序，避免修改时互相打架。具体顺序如下。

| 顺序 | 页面 | 原因 |
|---|---|---|
| 1 | 币种明细页 | 存在 P0 语义错误，必须先修正可信度问题 [3] |
| 2 | 钱包页 | 作为母页面，需要先稳住总资产与列表的统一基线 [2] |
| 3 | 聊天交易卡片页 | 在前两页基线稳定后继续做减法，统一服务单语气 [4] |
| 4 | 赚币页 | 主要是去展示稿痕迹，可在统一语言收口后同步完成 [5] |

## 五、Round 2 提交要求

Round 2 仍按仓库协作目录提交，只是继续采用“四页整包一起复核”的方式。正式提交前，每页都应完成以下自查：其一，是否还残留设计交付式说明文字；其二，是否还有语义不自洽字段；其三，是否仍有次要模块与主内容抢视线；其四，是否已经与其余三页保持统一风格和文案密度。[6]

## 六、当前执行结论

Round 2 已具备启动条件。下一步将直接基于本计划进入四页同步回改，并在完成后按仓库链路提交新的整包文件供总负责人统一复核。
