# 海豚社区 App：交易与赚币展示结构补充说明

**作者：Manus AI**  
**日期：2026-04-18**

## 一、最新产品口径的核心变化

基于你最新补充的要求，海豚社区 App 的结果展示结构需要进一步收口成两条非常清晰的主线。

第一条主线是**币种交易明细必须挂在对应币种下面**。如果用户做的是 USDT 相关操作，例如转出 USDT、用 USDT 去兑换其他资产，或者 USDT 被作为支出资产，那么这笔记录最合理的查看路径不是回聊天，也不是去一个泛化的总流水页，而是进入资产列表里的 **USDT**，再查看该币种下的交易明细。这样用户的检索路径最短，也最符合按资产记账的认知。

第二条主线是**赚币页面本身就是交易页面**。对于理财、申购、AI 策略投资这一类结果，不应该再混到普通币种流水里，而应该在赚币页面中承接。并且，只有在系统确实拿到了可展示的**真实投资产品**时，赚币页面才应该把产品展示给用户；如果没有真实产品，就应该直接显示空状态或不可用提示，而不是继续展示演示型产品卡片。

## 二、当前代码与新口径的主要偏差

从现有实现看，钱包页和赚币页都已经接近这个方向，但还没有完全对齐。

钱包页 `app/(tabs)/wallet.tsx` 的 `AssetRow` 当前只是静态展示资产余额、美元估值与价格来源，并**没有点击进入币种详情页的能力**。这意味着即便资产列表已经存在，用户仍然不能沿着“USDT → USDT 交易明细”这条自然路径去查看对应记录。

与此同时，赚币页 `app/(tabs)/earn.tsx` 已经具备一个较好的方向：它会调用 `searchDeFiProductsByMcp('USDT', 'ethereum')` 与 `searchDeFiProductsByMcp('ETH', 'ethereum')` 去加载真实产品，并把产品列表展示成策略卡片。这一点与“赚币页就是交易页”是相符的。但当前代码仍保留了 `fallbackStrategies` 这类**演示型兜底产品**。这和你现在的要求相冲突，因为你已经明确说了：**要有投资产品，才能展示出来给用户**。

| 模块 | 当前现状 | 与新口径的差距 | 建议调整 |
|---|---|---|---|
| 钱包页资产列表 | 只展示资产行，不可点击深入 | 用户无法从 USDT 等币种进入对应明细 | 让 `AssetRow` 变成可点击入口，进入币种详情页 |
| 币种详情承接 | 当前未见明确独立页面 | 无法按币种查看转账、兑换、收支记录 | 新增 `token-detail` 或类似路由，按币种展示记录 |
| 赚币页 | 已可拉取真实产品并展示策略卡片 | 仍保留 demo/fallback 策略，语义不纯 | 无真实产品时展示空状态，不再展示演示产品 |
| AI 策略结果展示 | 当前偏向“带入聊天继续确认” | 仍过度依赖聊天做下一步承接 | 赚币页直接作为策略交易页与结果页承接 |

## 三、建议的页面结构

如果按你的口径收口，我建议把页面职责分成下面两层，而不是再做一个统一的大流水页。

### 1. 币种详情页

币种详情页是从钱包资产列表点进来的第二层页面。它是**面向具体代币的交易明细页**，例如 USDT、ETH、BTC。这里重点展示的是与该币种直接相关的记录，包括转入、转出、兑换支出、兑换获得、手续费占用，以及该币种相关的失败记录。

| 字段 | 说明 |
|---|---|
| 币种 | 当前详情页所属币种，例如 USDT |
| 记录类型 | 转账、闪兑支出、闪兑收入、充值、提现等 |
| 时间 | 发起或记账时间 |
| 数量变化 | 增加或减少的数量 |
| 状态 | 处理中、成功、失败 |
| 摘要 | 例如“兑换 ETH 支出 100 USDT” |
| 明细标识 | `txHash` 或 `orderId`，仅作展开信息 |

这种设计的核心价值是：**用户找某个币的记录时，不需要先理解系统架构，只要点这个币就行。**

### 2. 赚币页 / 交易页

赚币页则是**投资产品与 AI 策略的交易承接页**。这里不是普通 token 流水页，而是“我有哪些可以投资的产品、我投了哪个产品、策略当前表现如何”的场所。它更像一个策略交易台，而不是单笔 token 明细页。

| 面板 | 建议展示内容 |
|---|---|
| 产品列表面板 | 产品名、协议、链、APR、TVL、风险等级、申购入口 |
| 已投策略面板 | 投入金额、当前净值、累计收益、今日收益、策略状态 |
| 运行解释面板 | 策略逻辑、资金分配、风险说明、适用市场环境 |
| 执行记录面板 | 申购成功、赎回成功、失败记录、更新时间 |

关键点在于：**只有当存在真实投资产品时，这个页面才展示产品卡片。** 如果 MCP 或后端没有返回真实产品，就应该直接给出“当前暂无可投资产品”的空状态，而不是用 demo 产品把页面填满。

## 四、对当前仓库最需要优先改的地方

从代码审查角度看，最优先的不是继续丰富聊天页，而是把下面两个入口补齐。

其一，是把钱包页中的 `AssetRow` 从纯展示组件改成可点击组件，让用户可以从资产总览真正走到某个币种的详情页。现在 `AssetRow` 只返回一个 `View`，没有交互能力，这正是“USDT 点下去看明细”这条路径尚未建立的直接证据。

其二，是把赚币页中的演示兜底策略删除或降级为空状态。当前 `app/(tabs)/earn.tsx` 在没有真实产品时会回退到 `ETH 稳健网格策略` 这类演示卡片，这与“要有投资产品才能展示出来给用户”的要求不一致。正确的做法应该是：真实产品存在时显示产品，不存在时显示空状态和刷新入口。

## 五、结论

所以，这轮最新产品口径可以收敛成一句非常明确的话：

> **普通交易跟着币走，投资策略跟着产品走。**

具体落地就是：

> **USDT 等币种的交易明细，挂在 USDT 详情页下面；赚币页就是交易页，但只有存在真实投资产品时才展示产品和策略数据面板。**

按这个标准回看当前仓库，方向已经接近，但还有两个关键缺口没有补齐：一是币种详情点击路径尚未建立，二是赚币页仍保留演示兜底产品。这两点正是后续最值得优先修正的地方。
