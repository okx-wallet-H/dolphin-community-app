# 海豚社区 App：逐页 UI 自审清单（Round 2）

**作者：Manus AI**  
**日期：2026-04-18**

## 一、本轮自审定位

本轮不是开放式重新设计，而是在已经推送的 MVP 基线上继续做 **高级感收口与层级清理**。我的处理原则仍然不变：先由我确定产品边界与视觉主次，再决定哪些点可以交给 UI 设计师辅助优化。也就是说，设计师后续只能参与“收口”，不能接管方向。

## 二、本轮已完成的收口动作

| 页面 | 本轮已处理 | 目的 |
|---|---|---|
| 钱包页 | 接入统一设计令牌、弱化快捷功能区、补上“链上资产”主区块标题 | 强化钱包首页的账户中台感 |
| 聊天交易卡片 | 修正浅色按钮文字对比度、压缩说明文案、降低工作台感 | 让聊天页更像服务入口而不是控制台 |
| 赚币页 | 接入统一设计令牌、压缩 hero 说明、降低洞察与快捷入口噪音 | 让真实产品列表成为页面主角 |
| 币种明细页 | 本轮主要保留，作为下一轮重点精修对象 | 避免同时多页发散改动 |

## 三、逐页审看结论

### 1. 钱包页

钱包页经过这轮处理后，已经比上一版更接近正式金融产品首页。主视觉区与资产列表之间的关系更清晰，快捷操作区不再过度抢眼，资产分组前增加了明确的“链上资产”标题，用户在浏览路径上会更自然地理解：**先看总览，再看具体币种，最后进入明细。**

不过，钱包页仍然保留了较多模块。`钱包安全` 与 `自动任务卡片` 仍然偏信息型，如果后续继续精修，我会优先处理这些模块的压缩表达，而不是先让设计师去重做 hero。

### 2. 聊天交易卡片

聊天页目前最重要的问题已经修掉一项：浅色幽灵按钮此前使用了深色背景按钮的白字样式，正式感和可读性都不够，本轮已修正为独立文案样式。与此同时，交换与转账卡片的结果提示也进一步缩短，不再说得过满，减少了“系统在这里做所有事”的错觉。

当前聊天页已经更加接近“发起与查看入口”，但仍未完全脱离旧工作台气质。下一轮若继续做，我会优先继续压缩指标区块和辅助文案，而不是增加新的卡片样式。

### 3. 赚币页

赚币页本轮完成的重点不是新功能，而是把视觉语言往统一系统拉回。页面已改用统一设计令牌作为核心样式来源，同时缩短了头部说明，让页面不再像长说明文档。Hero、洞察卡与快捷入口的关系也比上一轮更克制，产品列表开始重新成为视觉中心。

不过，赚币页还有一个明显事实：它现在仍包含较多辅助说明模块。如果进入下一轮，我会考虑进一步压缩 `insightRow` 和 `quickActionRow` 的存在感，甚至重新评估其中部分内容是否需要保留在首屏。

### 4. 币种明细页

币种明细页结构方向没有问题，因此本轮没有大动，只保留为下一轮重点精修对象。这是有意为之。因为如果钱包页、聊天页、赚币页同时在做主次关系压缩，币种明细页最好先保持稳定，避免整个系统四个页面同时漂移。

下一轮我准备重点处理该页的两件事：第一，进一步强化“状态标签—资产流向—详情标识”的阅读顺序；第二，把空状态、加载状态和正常记录态做出更强的一致性，让它真正像正式账本页，而不是刚新增的详情页。

## 四、我给 UI 设计师的页面级限制

| 页面 | 允许提出的建议 | 不允许提出的建议 |
|---|---|---|
| 钱包页 | hero 质感、资产行留白、分组节奏、卡片阴影细节 | 不允许改主入口，不允许把资产点击改成弹层 |
| 聊天页 | 按钮层级、卡片留白、状态区轻量化 | 不允许重新做成交易控制台 |
| 赚币页 | hero 节奏、辅助卡片的轻量化、真实产品卡片精修 | 不允许恢复 demo 卡片或演示型内容 |
| 币种明细页 | 账本卡排版、状态标签与摘要顺序、空状态质感 | 不允许把结果主入口改回聊天页 |
