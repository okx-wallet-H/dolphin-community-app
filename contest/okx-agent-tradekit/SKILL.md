# OKX Agent Trade Kit 交易赛参赛 SKILL

## Skill Name

**Contest Trend Trader for OKX Agent Trade Kit**

## Skill Purpose

该 Skill 面向 OKX Agent Trade Kit 交易赛，目标是在 **BTC-USDT-SWAP、ETH-USDT-SWAP、SOL-USDT-SWAP** 三个 USDT 永续合约之间，持续筛选出趋势结构最清晰、风险回报比最优的标的，并通过 Agent Trade Kit 执行真实交易。整个执行链路不直接调用交易 API，而是统一通过 Agent Trade Kit CLI 完成，以满足比赛记分要求。

## Reasoning Framework

该 Skill 并非依赖单一指标，而是采用 **技术结构、资金情绪、波动约束、账户风控** 的多因子联合推理框架。其核心思想是：先判断市场是否存在可交易趋势，再判断趋势是否具备足够的赔率与波动适配性，最后在账户层面确认当前回撤、资金费率与仓位预算是否允许执行。

| 推理层 | 输入 | 作用 |
| --- | --- | --- |
| 技术结构层 | 4h K线、EMA、RSI、MACD、ATR、布林带 | 判断方向、趋势强度与波动结构 |
| 情绪确认层 | 资金费率、持仓量 | 判断拥挤度、情绪偏向与持续性 |
| 波动约束层 | ATR、布林带带宽 | 避免过热追涨或过冷无效交易 |
| 风控执行层 | 仓位预算、杠杆、动态止损止盈 | 把研究结论转化为可执行订单 |

## Trigger Design

该 Skill 采用 **双触发机制**。第一类是 **定时触发**，即每 4 小时完成一次全市场扫描，用于建立大周期方向判断。第二类是 **事件驱动触发**，即在检测到已有仓位被止盈、止损或受控平仓后，立即重新分析三个标的，而不是机械等待下一个 4 小时周期。这一设计的目标是在趋势仍然存在时提高再入场效率。

## Data Collection Logic

在每次分析中，Skill 会针对三个目标永续合约拉取 4 小时级别行情，并计算以下指标：**EMA、RSI(14)、ATR(14)、MACD、布林带**。同时，Skill 还会采集 **资金费率** 与 **持仓量** 作为情绪和拥挤度的辅助判断依据。

## Decision Logic

Skill 会先对三个标的分别计算综合评分，再按趋势清晰度排序，最终只选择其中最强的一项执行。评分并不简单叠加，而是优先检查以下条件：

| 条件 | 处理原则 |
| --- | --- |
| 趋势不清晰 | 直接观望 |
| ATR 过低 | 视为无效波动，跳过 |
| ATR 过高 | 降低仓位，避免高波动误伤 |
| 资金费率绝对值过高 | 自动减半仓位，防止拥挤交易 |
| 多因子同时共振 | 允许开仓 |

当市场缺乏明确方向时，Skill 默认选择 **HOLD**。仅在人工测试模式下，才允许切换到 **受控功能测试单**，以验证交易链路本身是否正常，而不会将这种模式作为正式实盘逻辑长期启用。

## Position Sizing Logic

仓位管理采用 **风险预算优先** 的原则。单笔风险暴露默认不超过账户资金上限的既定比例；在本次测试阶段，进一步限制为 **不超过总资金 20%**。仓位大小会同时受到三个因素影响：第一，趋势信号强弱；第二，ATR 所代表的当前波动风险；第三，资金费率是否异常。

其结果是，越强的趋势允许越接近上限的预算，而 ATR 过高或资金费率异常时，系统会自动缩小仓位。

## Risk Control Logic

该 Skill 采用动态风险控制，而不是固定止损点数。止损距离由 **ATR** 决定，波动越大，止损越宽；波动越小，止损越紧。止盈则采用条件止盈与趋势延续思路结合的方式，在具备趋势延续迹象时给利润留空间。

此外，Skill 在账户层面还保留了以下保护逻辑：

| 风控项 | 说明 |
| --- | --- |
| 动态止损 | 基于 ATR 调整止损距离 |
| 条件止盈 | 下单时同步挂出止盈条件单 |
| 异常资金费率降仓 | 拥挤度过高时自动减仓 |
| 最大回撤保护 | 累计亏损触发后暂停新交易 |
| 事件驱动复核 | 仓位关闭后立即重新评估市场 |

## Execution Path

正式执行阶段统一通过 **OKX Agent Trade Kit CLI** 完成下单与查询。下单时会确保订单带有 **tag=agentTradeKit**，以满足交易赛排行榜记分要求。交易命令支持同步附带止损止盈条件单，从而在开仓瞬间完成基础保护。

## Validation Status

当前版本已经完成一次真实小仓位功能验证，验证内容包括：**订单创建、tag 校验、止损止盈挂载、仓位监控、受控平仓、事件驱动重分析**。测试结果表明，该 Skill 已具备进入持续观察运行阶段的基础能力。

## Operating Boundary

该 Skill 当前仅适用于以下范围：

1. 仅交易 **BTC-USDT-SWAP、ETH-USDT-SWAP、SOL-USDT-SWAP**。
2. 仅通过 **Agent Trade Kit** 执行，不直接调用交易 API。
3. 当前优先服务于比赛验证与低风险实盘测试，不追求高频与激进杠杆。
4. 正式扩大资金前，建议先继续观察多个 4 小时周期下的稳定性表现。
